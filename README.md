## Overview

This repository contains the full replication code for the empirical analysis in *Health Has No Price*, which estimates the causal effect of Brazil's *Farmácia Popular* drug subsidy program on preventable hospitalizations and mortality using a difference-in-differences event study design.

The identification strategy compares municipalities with high disease burden and low income (Treatment: Target group) against municipalities with low disease burden and low income (Reference group) around the 2013 program expansion.

## Replication Instructions

### Prerequisites

**R** (≥ 4.1) with the following packages:
```r
install.packages(c("data.table", "tidyverse", "duckdb", "DBI",
                   "jsonlite", "httr", "stringr", "readr"))
# microdatasus must be installed from GitHub:
remotes::install_github("rfsaldanha/microdatasus")
```

**Stata** (≥ 17) with the following packages (run once in Stata):
```stata
ssc install require,  replace
ssc install reghdfe,  replace
ssc install ftools,   replace
ssc install coefplot, replace
ssc install estout,   replace
ssc install gtools,   replace
```

### Additional Data Required (not on GitHub due to file size)

The RAIS formal employment microdata (~50 GB) must be obtained separately:

| Dataset | Source | Notes |
|---------|--------|-------|
| RAIS microdata | Ministério do Trabalho (MTE) | `.txt` files, semi-colon delimited, one file per state-year |
| SIH hospitalizations | DataSUS (`fetch_datasus`) | Downloaded automatically by `prepare_data.R` |
| SIM deaths | DataSUS (`fetch_datasus`) | Downloaded automatically by `prepare_data.R` |
| IBGE population | SIDRA API | Downloaded automatically by `prepare_data.R` |
| Race + literacy | `necessary_files/` | Already included in this repository |

---

### Step 1 — Run `prepare_data.R` in RStudio (once)

1. Open `prepare_data.R`
2. Edit the **PATHS** block at the top to match your machine:
   ```r
   RAIS_DIR <- "path/to/your/rais/txt/files"
   DB_PATH  <- "path/to/store/brazil_health_data_FULL.duckdb"
   CSV_OUT  <- "path/to/stata_sample/data/brazil_health_panel.csv"
   ```
   The `necessary_files/` paths do not need editing if you cloned the full repo.
3. Run the script (`source("prepare_data.R")` or click Source in RStudio)
4. Wait for the message: `=== ALL DONE. Run master.do in Stata next. ===`

The script builds the panel in six phases (SIH → SIM → Population → Demographics → RAIS → Assembly) and writes `data/brazil_health_panel.csv`. Each phase is skipped automatically on re-runs if its table already exists in DuckDB.

> **Note:** If you see `Failed to open database: file already in use`, close all other R sessions that may have DuckDB open and re-run.

---

### Step 2 — Run `master.do` in Stata

Open Stata, then run:
```stata
do "path/to/stata_sample/master.do"
```
Or open `master.do` in the Stata editor and click the Run button.

`master.do` runs the full pipeline in sequence and produces all tables and figures in `output/`.
